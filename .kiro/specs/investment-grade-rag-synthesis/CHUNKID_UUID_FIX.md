# ChunkId UUID Error Fix - SEC Filing Citations

## Problem

Error when storing citations from SEC filings:
```
Error: invalid input syntax for type uuid: "chunk-0"
```

## Root Cause

The `citations` table has foreign key constraints:
- `documentId` → references `documents(id)` (UUID)
- `chunkId` → references `document_chunks(id)` (UUID)

SEC filing citations from Bedrock don't have corresponding records in the `documents` or `document_chunks` tables. They use synthetic IDs like `"chunk-0"`, `"chunk-1"` which are NOT valid UUIDs.

### Citation Types

**User Document Citations:**
- Have valid `documentId` and `chunkId` UUIDs
- Reference actual records in the database
- Should be stored in `citations` table

**SEC Filing Citations (from Bedrock):**
- Have synthetic `chunkId` like `"chunk-0"`
- Don't have `documentId` (or it's null)
- Don't reference database records
- Should NOT be stored in database
- Should only be passed to frontend for display

## Solution

Filter citations before storing - only store user document citations with valid UUIDs:

```typescript
// Store citations for user documents ONLY
// SEC filing citations (from Bedrock) don't have documentId/chunkId in our DB
// They are passed to frontend for display but not stored
if (citations.length > 0) {
  // Filter to only user document citations (have valid documentId and chunkId UUIDs)
  const userDocCitations = citations.filter(c => 
    c.documentId && 
    c.chunkId && 
    // Check if it looks like a UUID (not "chunk-0" or similar)
    /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(c.chunkId)
  );
  
  if (userDocCitations.length > 0) {
    this.logger.log(`📎 Storing ${userDocCitations.length} user document citations`);
    
    const citationDtos = userDocCitations.map((citation) => ({
      tenantId,
      messageId: assistantMessage.id,
      documentId: citation.documentId,
      chunkId: citation.chunkId,
      quote: citation.excerpt || citation.snippet || '',
      pageNumber: citation.pageNumber,
      relevanceScore: citation.relevanceScore || citation.score,
    }));

    await this.citationService.createCitations(citationDtos);
    this.logger.log(`✅ Stored ${userDocCitations.length} user document citations`);
  }
  
  const secFilingCitations = citations.length - userDocCitations.length;
  if (secFilingCitations > 0) {
    this.logger.log(`ℹ️ Skipped storing ${secFilingCitations} SEC filing citations (passed to frontend only)`);
  }
}
```

## Why This Works

1. **User Document Citations** - Have valid UUIDs, get stored in database
2. **SEC Filing Citations** - Filtered out, NOT stored in database
3. **Frontend** - Receives ALL citations (both types) for display
4. **Modal** - Works for both types (doesn't need database records)
