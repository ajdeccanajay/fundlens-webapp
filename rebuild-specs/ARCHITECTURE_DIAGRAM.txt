KEY DIFFERENCES FROM CURRENT SYSTEM
═══════════════════════════════════════════════════════════════════════════════

CURRENT SYSTEM:                      PROPOSED SYSTEM:
─────────────────                    ────────────────────

1. Preprocessing                     1. Preprocessing
   ✗ Semantic chunking ALL content      ✓ Tables → Structured extraction
   ✗ Destroys table structure           ✓ Narratives → Fixed chunks
   ✗ 100 chunks retrieved               ✓ Metadata-rich storage

2. Retrieval                         2. Retrieval
   ✗ Vector search everything           ✓ Structured query for metrics
   ✗ Hope LLM finds right chunks        ✓ Metadata filters for precision
   ✗ No document routing                ✓ Intelligent doc routing (10-K vs 10-Q)

3. Response                          3. Response
   ✗ LLM extracts numbers from chunks   ✓ Pre-filled with exact values
   ✗ Hallucination possible             ✓ Zero hallucination on numbers
   ✗ Can't guarantee "latest" = both    ✓ Always returns Q + FY when needed
